# 动态规划 (Dynamic Programming, DP)

## 1. 核心前提：上帝视角 (Perfect Model)

要使用 DP 解强化学习问题，你必须知道环境的完整模型：

- **状态转移概率 $P(s'|s, a)$**：如果你在 $s$ 做 $a$，你精确知道去哪里的概率。
    
- **奖励函数 $R(s, a)$**：你精确知道每一步能拿多少钱。
    

**教授的类比：** 动态规划就像是拿着一张**完美的迷宫地图**在家里研究路线；而真正的“强化学习”则是把你蒙着眼丢进迷宫里去摸索。

---

## 2. 两个核心任务：预测与控制

DP 解决的问题可以拆解为两步：

### A. 策略评估 (Policy Evaluation) —— “预测”

- **目标**：给定一个策略 $\pi$，计算出它的价值函数 $V^\pi$。
    
- **方法**：把贝尔曼方程变成一个**迭代更新**的公式：
    
    $$V_{k+1}(s) = \sum_{a} \pi(a|s) \sum_{s', r} p(s', r|s, a) [r + \gamma V_k(s')]$$
    
- **直觉**：通过不断地“刷数据”，让每个点的分数值根据邻居的分数不断演进，直到最后数值不再变化（收敛）。
    

### B. 策略改进 (Policy Improvement) —— “变得更好”

- **目标**：如果你已经知道了当前策略的价值 $V^\pi$，如何变强？
    
- **方法**：**贪心 (Greedy)**。在每个状态下，看看哪个动作能带来最大的 $Q(s, a)$，就选那个。
    
    $$\pi'(s) = \arg\max_a \sum_{s', r} p(s', r|s, a) [r + \gamma V(s')]$$
    

---

## 3. 两大经典算法

### 3.1 策略迭代 (Policy Iteration)

这是一个“两步走”的循环：

1. **评估**：算出当前的 $V$。
    
2. **改进**：根据 $V$ 得到更好的 $\pi$。
    
3. **重复**：直到策略不再改变。
    

> **评价**：比较慢，因为每次改进前都要把 $V$ 算得很准，但通常只需要很少的步数就能找到最优策略。

### 3.2 价值迭代 (Value Iteration)

这是一个“一步到位”的方法。它不等 $V$ 完全算准，而是每次只更新一点点，同时直接在公式里取 $\max$：

$$V_{k+1}(s) = \max_a \sum_{s', r} p(s', r|s, a) [r + \gamma V_k(s')]$$

> **评价**：直接解贝尔曼最优方程。计算效率更高，是很多现代算法的雏形。

---

## 4. 动态规划的局限性

虽然 DP 很完美，但在现实中它面临两大杀手：

1. **维数灾难 (Curse of Dimensionality)**：如果状态太多（比如围棋有 $10^{170}$ 个状态），你根本没法存下那张 $V$ 表。
    
2. **模型依赖 (Model Dependency)**：现实中我们几乎永远不知道精确的 $P$ 和 $R$。我们不知道自动驾驶时，下一秒旁边的车会怎么开。
    

---

## 5. 教授的深度总结：DP 是 RL 的“灵魂导师”

虽然我们现在很少直接用 DP 解决复杂的 AI 问题，但它是所有强化学习算法的**理论母体**：

- **蒙特卡洛 (MC) 方法**：是在没地图的情况下，通过“跑完全程”来估算 DP 里的积分。
    
- **时序差分 (TD) 学习**：是把 DP 的迭代思想和采样结合起来，边走边算。
    

> **理解透的标志**：你能意识到，**所谓的“学习”，本质上就是当没有地图（$P$ 和 $R$ 未知）时，我们如何通过经验来逼近动态规划的结果。**
